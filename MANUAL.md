## 1. Введение

**AdaptiveQuantGrid (AQG)** — профессиональная адаптивная сеточная торговая система для MetaTrader 5.

### Ключевые особенности

| Функция | Описание |
|---------|----------|
| 🔄 **Адаптивная сетка** | Шаг сетки автоматически подстраивается под волатильность через ATR |
| 🧭 **Фильтр режима рынка** | ADX + EMA определяют: флэт, восходящий или нисходящий тренд |
| 🛡️ **Многоуровневый риск** | Просадка, глубина сетки, профиль риска, маржинальный контроль |
| 🔥 **Smart Overlap** | Автоматическое закрытие прибыльных пар позиций (лучший Buy + худший Sell) |
| 📈 **Equity Trailing** | Трейлинг по общей прибыли корзины с фиксацией пика |
| 💾 **Recovery System** | Восстановление состояния после перезапуска терминала |
| 🔄 **Smart Order Trail** | Автоматический перенос сетки при сильном отклонении цены |
| 📊 **UI Panel** | Информационная панель на графике в реальном времени |

## 2. Архитектура системы

EA построен по модульному принципу. Каждый модуль отвечает за свою подсистему:

```
AdaptiveQuantGrid.mq5  (Главный контроллер)
├── Config.mqh          — Конфигурация, enum-ы, input-параметры
├── Logger.mqh          — Логирование с уровнями и троттлингом
├── TradeExecutor.mqh   — Исполнение ордеров через CTrade
├── RiskGuardian.mqh    — Управление рисками
├── GridCalculator.mqh  — Расчёт уровней сетки, ATR-шаг, размещение ордеров
├── RegimeDetector.mqh  — Определение рыночного режима (ADX + EMA)
├── GridDirectionEngine.mqh — Направленная логика (MODE_LEGACY)
├── RecoveryManager.mqh — Восстановление после перезапуска
├── OverlapEngine.mqh   — Умное перекрытие убытков
├── EquityTrailer.mqh   — Трейлинг по корзине позиций
└── UI_Panel.mqh        — Информационная панель на графике
```

### Жизненный цикл EA

```
[OnInit] → Валидация среды → Инициализация модулей → Сканирование позиций
     ↓
  [STATE_IDLE] → PlaceInitialOrders → [STATE_ACTIVE]
     ↓
  [OnTick] → Обновление режима → Обновление сетки → Overlap → Trailing
     ↓
  [OnTradeTransaction] → Обработка сделок → DirectionEngine (LEGACY)
     ↓
  [OnDeinit] → Сохранение состояния → Очистка панели
```

## 3. Установка

### Способ 1: Из исходного кода

1. Откройте **MetaEditor** (F4 в терминале)
2. Скопируйте папку `SOURCE/` в:  
   `MQL5/Experts/AdaptiveQuantGrid/`
3. Откройте `AdaptiveQuantGrid.mq5` и нажмите **F7** (Компилировать)
4. Файл `AdaptiveQuantGrid.ex5` появится в `MQL5/Experts/`

### Способ 2: Готовый файл

1. Скопируйте `AdaptiveQuantGrid.ex5` в:  
   `MQL5/Experts/`
2. Перезапустите MetaTrader 5

### Запуск на графике

1. Откройте нужный символ и таймфрейм
2. Перетащите `AdaptiveQuantGrid` из навигатора на график
3. Настройте параметры → вкладка **"Параметры"**
4. Нажмите **OK**

## 4. Режимы торговли

### 4.1 MODE_SYMMETRIC — Симметричная сетка

**Принцип работы:**
- На каждом уровне выставляются **оба ордера**: Buy Limit + Sell Limit
- Нет направления — сетка работает в обе стороны
- Подходит для **флэтового** рынка

**Схема:**
```
Цена:                    ────────────────── Якорь (Anchor)
                         │
Уровень +1:              ├──── Sell Limit
                         │
Уровень +2:              ├──── Sell Limit
                         │
Уровень -1:              ├──── Buy Limit
                         │
Уровень -2:              ├──── Buy Limit
```

### 4.2 MODE_LEGACY — Асимметричный направленный режим

**Принцип работы:**
- При старте: **Market Buy + Market Sell + Buy Limit**
- При движении **вниз** (срабатывает Buy Limit):
  - Закрывается текущий Market Sell
  - Открывается новый Market Sell на текущем уровне
  - Выставляется Buy Limit на следующий уровень
- При движении **вверх** (срабатывает SL на Market Sell):
  - Частично закрывается лучший Buy
  - Открывается новый Market Sell
  - Выставляется Buy Limit меньшего объёма

**Схема:**
```
Якорь: ──────────────────────────────
                              Market Buy (базовый)
                              Market Sell (базовый, со SL)
                              Buy Limit (уровень 1)
                                           │
                              При срабатывании ↓
                                           │
                              Market Sell закрывается
                              Новый Market Sell ниже
                              Buy Limit на уровень 2
```

## 5. Подробное описание параметров

### 5.1 🎯 Core Settings — Основные настройки

| Параметр | Тип | По умолч. | Описание |
|----------|-----|-----------|----------|
| `InpGridMode` | ENUM_AQG_GRID_MODE | MODE_LEGACY | **Режим торговли:** `MODE_SYMMETRIC` — симметричная сетка, `MODE_LEGACY` — асимметричный направленный режим |
| `InpGridAnchorOffset` | double | 0.0 | **Смещение якоря** в пунктах от текущей цены. Если > 0, якорь смещается на указанное количество пунктов |
| `InpATRPeriod` | int | 14 | **Период ATR** (Average True Range). Используется для расчёта динамического шага сетки. Больше → плавнее реакция |
| `InpATRTF` | ENUM_TIMEFRAMES | PERIOD_H1 | **Таймфрейм ATR**. Рекомендуется H1 для внутридневной торговли, H4 для среднесрочной |
| `InpATRMultiplier` | double | 1.4 | **Множитель ATR**. Шаг сетки = ATR × этот множитель. Больше → шире шаг → реже срабатывания |
| `InpMinStepPoints` | double | 150.0 | **Минимальный шаг** в пунктах. Ограничивает снизу динамический шаг. Защита от слишком частой сетки |
| `InpMaxStepPoints` | double | 350.0 | **Максимальный шаг** в пунктах. Ограничивает сверху. Защита от слишком редкой сетки |
| `InpMaxGridDepth` | int | 10 | **Максимальная глубина** сетки (количество уровней). При достижении новые уровни не добавляются |

### 5.2 📦 Lot Management — Управление лотами

| Параметр | Тип | По умолч. | Описание |
|----------|-----|-----------|----------|
| `InpUseAsymmetricLots` | bool | true | **Асимметричные лоты**. Если true, Buy и Sell используют разные базовые объемы (только в MODE_LEGACY) |
| `InpBuyBaseLot` | double | 0.03 | **Базовый лот Buy**. Объем для Buy-позиций и Buy Limit ордеров |
| `InpSellBaseLot` | double | 0.01 | **Базовый лот Sell**. Объем для Market Sell в режиме MODE_LEGACY |
| `InpPartialCloseLot` | double | 0.02 | **Лот частичного закрытия**. Объем для частичного закрытия при движении вверх в MODE_LEGACY |
| `InpSellSL_Points` | double | 250.0 | **Stop Loss для Market Sell** в пунктах. Расстояние от цены открытия до SL |
| `InpMinLotSize` | double | 0.01 | **Минимальный лот**. Нижняя граница объема (обычно 0.01) |
| `InpMaxLotSize` | double | 0.50 | **Максимальный лот**. Верхняя граница объема (защита от чрезмерного увеличения) |

### 5.3 🛡️ Risk Management — Управление рисками

| Параметр | Тип | По умолч. | Описание |
|----------|-----|-----------|----------|
| `InpRiskProfile` | ENUM_AQG_RISK_PROFILE | PROFILE_BALANCED | **Профиль риска:** `CONSERVATIVE` — консервативный (×0.5 лот, 20% DD, 8 уровней), `BALANCED` — сбалансированный (×1.0, 40% DD, 10 уровней), `AGGRESSIVE` — агрессивный (×2.0, 60% DD, 14 уровней) |
| `InpRiskPerTradePercent` | double | 0.35 | **Риск на сделку** в процентах от баланса. Умножается на множитель профиля риска |
| `InpMaxEquityDrawdownPercent` | double | 20.0 | **Максимальная просадка эквити** в процентах. При достижении — аварийная остановка |
| `InpUseDynamicLot` | bool | false | **Динамический расчёт лота**. Если true, лот рассчитывается на основе доступной маржи (резерв на будущее) |

### 5.4 🧭 Market Regime Filter — Фильтр рыночного режима

| Параметр | Тип | По умолч. | Описание |
|----------|-----|-----------|----------|
| `InpUseRegimeFilter` | bool | true | **Включить фильтр режима**. Если true, EA определяет текущий режим рынка через ADX + EMA |
| `InpADXPeriod` | int | 14 | **Период ADX**. Больше → плавнее, но медленнее реакция на изменение волатильности |
| `InpADXThreshold` | double | 25.0 | **Порог ADX**. Если ADX < порога → FLAT, если >= → TREND |
| `InpEMAPeriod` | int | 50 | **Период EMA**. Используется для определения направления тренда (цена > EMA → UP, цена < EMA → DOWN) |
| `InpRegimeHysteresis` | int | 3 | **Гистерезис в барах**. Количество баров подтверждения перед сменой режима. Защита от ложных переключений |

**Как работает фильтр:**
```
ADX < 25  →  MODE_FLAT      →  Шаг сетки × 0.7  (сужается)
ADX >= 25 + Цена > EMA  →  MODE_TREND_UP    →  Шаг сетки × 1.3  (расширяется)
ADX >= 25 + Цена < EMA  →  MODE_TREND_DOWN  →  Шаг сетки × 1.3  (расширяется)
```

### 5.5 💰 Cycle Exit Strategy — Стратегия выхода из цикла

| Параметр | Тип | По умолч. | Описание |
|----------|-----|-----------|----------|
| `InpUseLegacyExit` | bool | true | **Классический выход по прибыли**. Если true, при достижении `InpTargetProfit` все позиции закрываются и цикл начинается заново |
| `InpTargetProfit` | double | 60.0 | **Целевая прибыль** в долларах. При достижении — закрытие всех позиций |
| `InpUseEquityTrailing` | bool | false | **Трейлинг корзины**. Если true, вместо фиксированного таргета используется трейлинг с фиксацией пика |
| `InpTrailingActivationProfit` | double | 45.0 | **Прибыль активации трейлинга** ($). Трейлинг включается только при достижении этого уровня |
| `InpTrailingDropPercent` | double | 22.0 | **Порог отката** (%). При откате от пика на этот процент — закрытие всех позиций |
| `InpTrailingCheckSec` | int | 12 | **Интервал проверки** в секундах. Как часто проверяется состояние корзины |

**Рекомендация:** Используйте **или** `InpUseLegacyExit` **или** `InpUseEquityTrailing`, но не оба одновременно. Trailing лучше подходит для трендового рынка, Legacy Exit — для флэта.

### 5.6 🔥 Smart Overlap Engine — Умное перекрытие

| Параметр | Тип | По умолч. | Описание |
|----------|-----|-----------|----------|
| `InpUseSmartOverlap` | bool | true | **Включить Overlap Engine**. Если true, EA периодически ищет пары позиций для прибыльного закрытия |
| `InpOverlapProfitTarget` | double | 25.0 | **Целевая прибыль перекрытия** ($). Суммарная прибыль пары должна быть >= этого значения |
| `InpOverlapCooldownMin` | int | 5 | **Кулдаун в минутах**. Минимальное время между перекрытиями. Защита от частых закрытий |

**Как работает Overlap:**
```
1. Собираем: Лучший Buy, Худший Buy, Лучший Sell, Худший Sell
2. Пробуем пару #1: Лучший Buy + Худший Sell → Сумма >= Target? → Закрываем
3. Пробуем пару #2: Лучший Sell + Худший Buy → Сумма >= Target? → Закрываем
4. Кулдаун 5 минут перед следующим поиском
```

### 5.7 🔄 Smart Order Relocation — Умный перенос ордеров

| Параметр | Тип | По умолч. | Описание |
|----------|-----|-----------|----------|
| `InpUseSmartOrderTrail` | bool | true | **Включить перенос**. Если true, сетка автоматически переносится при сильном отклонении цены |
| `InpOrderTrailDistance` | double | 450.0 | **Дистанция триггера** в пунктах. При отклонении цены от якоря на это расстояние → перенос сетки |

**Как работает Smart Trail:**
```
1. |Цена - Якорь| > 450 пунктов?
2. Цена НЕ приближается к ближайшим лимиткам? (проверка 50% от Trigger Distance)
3. Если ДА → Удаляем все pending-ордера, Якорь = текущая цена, шаг = 0
4. В следующем OnTick выставляются новые ордера от нового якоря
```

## 6. Как работает EA

### 6.1 Инициализация (OnInit)

```
1. Рассчитывается Magic Number (AQG_MAGIC_BASE + символ)
2. Инициализируется Logger
3. Проверяется среда (подключение к серверу, торговля разрешена)
4. Инициализируются все модули:
   ├── TradeExecutor
   ├── RiskGuardian (профиль риска, проверка просадки)
   ├── GridCalculator (ATR-шаг, лимиты, якорь)
   ├── RegimeDetector (ADX, EMA)
   ├── RecoveryManager
   ├── OverlapEngine
   ├── EquityTrailer
   ├── DirectionEngine (только MODE_LEGACY)
   └── UI_Panel
5. Сканируются открытые позиции:
   ├── Есть позиции → STATE_ACTIVE, восстанавливается якорь
   └── Нет позиций → STATE_IDLE, ждём OnTick для размещения
6. Загружается состояние из файла (если InpSaveStateToFile = true)
```

### 6.2 Цикл OnTick

```
Каждый тик:
├── Проверка Emergency Stop
├── Проверка просадки (RiskGuardian)
├── Legacy Exit Check (каждые 60 секунд):
│   └── BasketProfit >= TargetProfit → Закрыть всё → STATE_IDLE → Новый цикл

Каждые 2 секунды:
├── STATE_IDLE? → PlaceInitialOrders → STATE_ACTIVE
├── Обновить рыночный режим (RegimeDetector)
├── Рассчитать множитель шага (flat ×0.7, trend ×1.3)
├── Обновить сетку (новые уровни)
├── Smart Order Trail (перенос при отклонении)
├── Smart Overlap (поиск прибыльных пар)
└── Equity Trailing (если включён и не Legacy Exit)
```

### 6.3 Обработка сделок (OnTradeTransaction)

```
Только MODE_LEGACY:
├── TRADE_TRANSACTION_DEAL_ADD
├── Проверка Magic, Symbol
├── Событие DOWNWARD (сработал Buy Limit):
│   ├── Увеличить шаг на +1
│   ├── Закрыть текущий Market Sell
│   ├── Открыть новый Market Sell ниже
│   └── Выставить Buy Limit на следующий уровень
└── Событие UPWARD (сработал SL на Market Sell):
    ├── Уменьшить шаг на -1
    ├── Частично закрыть лучший Buy
    ├── Открыть новый Market Sell
    └── Выставить Buy Limit меньшего объёма
```

## 7. Управление рисками

### 7.1 Профили риска

| Профиль | Множитель лота | Макс. просадка | Макс. глубина |
|---------|----------------|----------------|---------------|
| CONSERVATIVE | ×0.5 | 20% | 8 уровней |
| BALANCED | ×1.0 | 40% | 10 уровней |
| AGGRESSIVE | ×2.0 | 60% | 14 уровней |

**Множитель влияет на:**
- `InpRiskPerTradePercent` умножается на множитель
- Максимальная допустимая просадка
- Максимальная глубина сетки

### 7.2 Аварийная остановка

Срабатывает при:
- Просадка эквити >= `InpMaxEquityDrawdownPercent`
- Терминал отключён от сервера
- Торговля запрещена на счёте

**Действия:**
1. `STATE_EMERGENCY`
2. Закрытие ВСЕХ позиций с Magic EA
3. Удаление ВСЕХ pending-ордеров
4. Push-уведомление (если включено)
5. EA перестаёт торговать до перезапуска

### 7.3 Маржинальный контроль

Перед каждым открытием позиции:
```
1. Рассчитать требуемую маржу
2. Проверить свободную маржу
3. Если FreeMargin < Margin × 1.5 → Отказ от сделки
```

### 7.4 Лимит глубины сетки

```
Текущий шаг >= Макс. глубина → Новые уровни НЕ добавляются
Существующие позиции продолжают работать
```

## 8. Система восстановления состояния

### 8.1 Сохранение состояния

При деинициализации (`OnDeinit`) сохраняется:
- Текущий рыночный режим
- Состояние сетки (STATE_IDLE / ACTIVE / и т.д.)
- Цена якоря
- Magic Number

**Файл:** `AQG_State_<Symbol>_<Magic>.dat` в `MQL5/Files/`

### 8.2 Восстановление

При инициализации (`OnInit`):
```
1. Загрузка из файла (если InpSaveStateToFile = true)
2. Если файл не найден → RecoveryManager:
   ├── Сканирование открытых позиций
   ├── Сканирование pending-ордеров
   ├── Реконструкция якорной цены (среднее между BL_1 и SL_1)
   ├── Определение максимального шага
   └── Валидация (цена не слишком далеко от текущей)
3. Синхронизация шага по реальным позициям
4. Если позиции есть → STATE_ACTIVE
5. Если позиций нет → STATE_IDLE
```

### 8.3 Валидация восстановления

```
|Текущая цена - Якорь| <= Шаг × InpMaxStepPoints × 1.5
```

Если валидация не пройдена → Позиции и ордера закрываются → EA стартует с нуля.

## 12. Часто задаваемые вопросы

### Q: Почему EA не открывает позиции?

**A:** Возможные причины:
1. `STATE_IDLE` — нет позиций, EA ждёт размещения начальных ордеров
2. `STATE_EMERGENCY` — сработала аварийная остановка (просадка)
3. Недостаточно свободной маржи
4. Достигнута максимальная глубина сетки
5. Рыночный фильтр запрещает направление (только FLAT, TREND_UP, TREND_DOWN)

### Q: Как сбросить состояние EA?

**A:** 
1. Удалите файл `AQG_State_<Symbol>_<Magic>.dat` из `MQL5/Files/`
2. Перезапустите EA
3. Или закройте все позиции вручную → EA автоматически сбросится в `STATE_IDLE`

### Q: Можно ли использовать на нескольких символах?

**A:** Да. Magic Number автоматически генерируется на основе символа, поэтому экземпляры не пересекаются.

### Q: Как работает Smart Order Trail?

**A:** Если цена отклоняется от якоря на `InpOrderTrailDistance` пунктов и **не приближается** к ближайшим лимиткам — все pending-ордера удаляются, якорь переносится на текущую цену, сетка перестраивается.

### Q: Что делать при Emergency Stop?

**A:** 
1. Проверьте просадку эквити в журнале
2. Проанализируйте причины (глубокая сетка, сильный тренд)
3. При необходимости перезапустите EA (состояние сбросится)
4. Рассмотрите снижение `InpMaxGridDepth` или изменение профиля риска

### Q: Чем отличаются Legacy Exit и Equity Trailing?

**A:** 
- **Legacy Exit** — фиксированный таргит прибыли ($). При достижении — закрытие ВСЕХ позиций сразу.
- **Equity Trailing** — динамический выход. Фиксирует пик прибыли, закрывает при откате X%. Позволяет "поймать" больший профит в тренде.

### Q: Почему Overlap не срабатывает?

**A:** 
1. Кулдаун ещё не истёк (`InpOverlapCooldownMin`)
2. Нет пар с суммарным профитом >= `InpOverlapProfitTarget`
3. Только одна сторона присутствует (только Buy или только Sell)

### Q: Можно ли запустить на VPS?

**A:** Да. Рекомендуется:
- Включить `InpSaveStateToFile = true`
- Убедиться в стабильном подключении к серверу
- Настроить автозапуск терминала на VPS
